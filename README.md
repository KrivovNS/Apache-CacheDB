### Быстрый старт

По умолчанию сервер запускается на порту 7070.
При инициализации автоматически создается пользователь `default` с паролем `admin123` и правами
`superadmin`. 

**Важно: смените пароль для пользователя `default` до начала работы с хранилищем!**

Пользователь `deafult` обладает правами на все команды. Чтобы сделать сменить пароль, выполните запрос:

GET http://localhost:8080/auth?login=default&password=admin123 

в ответе от сервера вы получите токен сессии.
Используйте данный токен в запросе:

PUT http://localhost:8080/user?session_token={полученый_токен}login=default&password={новый_пароль}.

Поздравляю, Вы изменили пароль default пользователя! Пересоздайте сессию `default` юзера и теперь вы
можете пользоваться всеми возможностями Apache CacheDB!

### Api запросов

Сервер работает на протоколе http. Структура запроса:

- Метод
- URI: в формате {host}/{req_type}?{parameters}
    - `host` по умолчанию `localhost:7070`
    - `req_type` - один из пяти видов запросов
    - `parameters` - параметры в формате {parameter_name}={value} разделенные `&`
- Тело запроса(нужно лишь для методов POST PUT при обращении кэшу)

Виды запросов(`req_type`):
- `/` : выполняется с методом GET, возвращается информационная страница с описанием запросов
- `/auth` : выполняется с методов GET, в качестве параметров нужно указать `login`, `password` 
при успешной верификации сервер вернет токен сессии необходимый для распознавания пользователя системой
- `/configuration` : выполняется с методом PUT, до выполнения 1 запроса к кэшу! 
Параметры: `session_token`, `max_memory_policy`, `max_storage_memory`, `persistence`
  - `session_token` - передается токен сессии `default` юзера
  - `max_memory_policy` - передается название 1 из политик вытеснения кэша:
    - `noeviction` - пользователь сам управляет памятью, при достижении лимита приходит оповещение о невозможности сохранения данных
    - `allkeys-lru` - при достижении лимита памяти система автоматически среди всех ключей по принципу LRU удаляет данные, до тех пор пока не освободится достаточно места для выполнения запроса
    - `volatile-lru` - при достижении лимита памяти система автоматически сначала среди ключей с `ttl`, потом среди ключей без `ttl` по принципу LRU удаляет данные, до тех пор пока не освободится достаточно места для выполнения запроса
    - `allkeys-lfu` - при достижении лимита памяти система автоматически среди всех ключей по принципу LFU удаляет данные, до тех пор пока не освободится достаточно места для выполнения запроса
    - `volatile-lfu` - при достижении лимита памяти система автоматически сначала среди ключей с `ttl`, потом среди ключей без `ttl` по принципу LFU удаляет данные, до тех пор пока не освободится
  - `max_storage_memory` - в байтах указывается объем памяти выделяемый под хранение данных
  - `persistence` - нужно ли обеспечить персистентность данных, укажите `true` или `false`
- `/user` (только для `default` пользователя):
  - POST: создание нового пользователя. Параметры: `session_token`, `login`, `password`, `permission`
    - `permission`: нужно указать вид прав пользователя: `admin` (все операции с кэшами), `reader` - только чтение данных. 
  - PUT: обновление пользователя. Параметры:
    - Обязательные `session_token`, `login`(логин существующего пользователя)
    - Новые данные пользователя(нужно передать хотя бы 1 из параметров: `new_login`, `password`
  - DELETE: удаление пользователя. Параметры: `session_token`, `login`
- `/cache`:
  - GET: получение данных. Параметры: `session_token`, `key`
  - POST: сохранение данных. Параметры: `session_tokem`, `key`, `type`, `ttl` в теле запроса указываются данные для сохранения
    - Доступные типы: `string`, `json`, `byte[]`
    - `ttl` - необязательный параметр, указывается время жизни данных с этим ключом.
  - PUT: обновление данных(ключ должен существовать). Параметры: `session_token`, `key`, `type`, `ttl`
  - DELETE: удаление данных по ключу. Параметры: `session_token`, `key`
